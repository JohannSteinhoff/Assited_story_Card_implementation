success, and tells the user whether signup worked or failed. No login, no
email verification, no OAuth — just the one form completing the one action.

AC3 — Password must be at least 8 characters

   - short password (< 8 chars)
   - password mismatch
   - duplicate email

3. POST /api/signup controller
   - parses request body

   - returns appropriate HTTP status + message

4. API-level tests covering:

FILE CREATED: backend/src/services/AuthService.ts
---------------------------------------------------
  type SignupResult = { ok: boolean; error?: string }

  interface User { email: string; passwordHash: string }

  class AuthService:
    private users: User[] = []

    async signup(email, password, confirmPassword): Promise<SignupResult>
      1. regex check email           → { ok: false, error: "Invalid email address" }
      2. password.length < 8         → { ok: false, error: "Password must be at least 8 characters" }
      3. password !== confirmPassword → { ok: false, error: "Passwords do not match" }
      4. duplicate email check        → { ok: false, error: "That email is already registered" }
      5. bcrypt.hash(password, 10)   → push user → { ok: true }

CONFIG CHANGES
---------------
  tsconfig.json: added "esModuleInterop": true  (required for bcrypt default import)
  SignupResult type: flattened to { ok: boolean; error?: string }
    (discriminated union was too strict for test access without narrowing)

CHOSEN REFACTOR: extract validate() from handleSubmit in SignupForm.tsx

WHY THIS ONE
-------------
  - Three validation guards were inline inside an async event handler,
    mixing pure logic with side effects (setError, return)
  - Extracting them into a pure function makes handleSubmit read as three
    clean phases: validate → fetch → update state
  - Purely internal to one file — no interface changes, no new deps,
    tests don't care about internal structure

CHANGE (SignupForm.tsx only)
-----------------------------
  ADDED above component:
    function validate(email, password, confirmPassword): string | null
      if invalid email  → return "Invalid email address"
      if password < 8   → return "Password must be at least 8 characters"
      if mismatch       → return "Passwords do not match"
      return null

  REPLACED in handleSubmit:
    BEFORE: 9 lines of inline if-guards with setError + return
    AFTER:  const validationError = validate(...)
            if (validationError) { setError(validationError); return; }
